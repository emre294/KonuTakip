# KonuTakip Android BuildConfig

For this Expo SDK 54 / React Native 0.81 app, `BuildConfig` references in the generated
Kotlin entry points require `buildFeatures { buildConfig true }` in the Android app
module. Because `android/` is ignored and regenerated by Expo, keep that setting in an
Expo config plugin applied from `app.json`, not only in the generated Gradle file.

**Why:** A clean prebuild replaces ignored native files, so a manual edit to
`android/app/build.gradle` alone is not durable.

**How to apply:** Keep `android.namespace`, `defaultConfig.applicationId`, and both
Kotlin file package declarations aligned with the Play Console application ID.
