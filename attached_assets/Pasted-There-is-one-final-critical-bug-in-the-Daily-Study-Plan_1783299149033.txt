==================================================
BUG
==================================================

Current behavior:

A recurring daily study session is completed by pressing
"Complete Session".

The session is successfully completed.

However, on the following day the application still reports:

"You have 1 overdue study session."

This is incorrect.

==================================================
ROOT CAUSE
==================================================

The overdue calculation is treating recurring sessions as permanently overdue based only on their original scheduled date.

A recurring session must be evaluated PER OCCURRENCE (per day), not by its original creation date.

==================================================
EXPECTED BEHAVIOR
==================================================

Example:

Recurring session
Every day

Monday:
User completes it.

Tuesday:
Monday's occurrence must NEVER be considered overdue.

Only Tuesday's occurrence exists now.

No overdue warning.

--------------------------------------------------

Example:

Monday:
User does NOT complete it.

Tuesday:
Show 1 overdue session.

==================================================
IMPLEMENTATION
==================================================

Audit the overdue calculation.

For recurring sessions:

Do NOT compare today's date with the original session date.

Instead determine whether the PREVIOUS OCCURRENCE was completed.

If the previous occurrence was completed,
exclude it from overdue results.

Only unfinished occurrences may be counted as overdue.

✓ Statistics remain unchanged
